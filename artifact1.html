Artifact 1: Toy Store Management System

Objective: The goal of this project was to develop a Toy Store Management System that allows users to find, purchase, and manage a database of toys. This project leveraged object-oriented programming (OOP) principles, including inheritance, abstraction, and interfaces.

Key Features

  • Search inventory by serial number, name, or type with case-insensitive search functionality.
  • Purchase toys while automatically updating inventory counts.
  • Add or remove toys from the database with unique validation for serial numbers.
  • Save and load toy data to/from a text file, maintaining persistence across program runs.
  • Bonus Feature: Gift suggestion based on age, type, or price range.

Challenges and Solutions

Challenge: Implementing OOP structure effectively to model toys and their behaviors.
Solution: Designed an abstract superclass `Toy` and derived classes (e.g., `Figure`, `Animal`, `Puzzle`, `BoardGame`) to manage unique attributes for each toy type.

Challenge: Ensuring data consistency when multiple operations modified the inventory.
Solution: Centralized all toy data in an `ArrayList` and used appropriate synchronization for thread safety.

Challenge: Validating user inputs for toy creation and transactions.
Solution: Added detailed input validation and custom exception handling (e.g., invalid serial numbers, negative prices).

Workflow

  • Design Phase: Created a project structure based on MVC architecture and documented a UML diagram for OOP hierarchy.
  • Development Phase: Used Eclipse IDE and Git for version control, dividing tasks between team members to ensure balanced contributions.
  • Testing Phase: Developed JUnit tests for key functionality, including toy search, inventory updates, and exception handling.
